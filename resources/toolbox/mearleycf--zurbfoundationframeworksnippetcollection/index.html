Zurb Foundation Framework Snippet Collection

Zurb Foundation Framework - Snippets for Komodo Edit

Download

Download Source

Hi!

This simple git repository stores komodo edit snippets i've created in order to make development of the zurb foundation framework easier.

Contents: 

+ Containers & Columns
- 1_column.komodotool

Usage: double-click the snippet to add it to your page.  You will be prompted to enter one or more variables.  Basically, you're setting up your framework DIV containers.  If you use 1_column, you are saying "I only want my content to be spaced in to one column."  The variable you enter is the width of your ONE column--so you can make it one column wide ("one"), etc.  Up to twelve, I believe.  

You need to use the actual text of the number of columns, because this is how the CSS file is set up by curb.  

Example:

entering "one" will generate: 

<div class="one columns">

</div><!--one columns-->

- 2_columns.komodotool

Usage: same as above.  The difference here is that you're setting up two columns, and there are two variables--the # of columns wide each of your two columns should be.

Example: 
|  column A is six columns wide  |  column B is 4 columns wide |

You would enter "six" for the first column variable, and "four" for the second column variable. 

Your output, then, is: 

<div class="six columns">

</div>
<div class="four columns">

</div><!--six & four columns--> 

And this produces in the UI a 10-column wide setup, split into 2 DIV containers, with a 60/40 split of that 10 column width.  

- 3_columns.komodotool
- 5_columns.komodotool
- 6_columns.komodotool
- base_generic_html.komodotool

It is the required <head> section including all CSS file references, <title>, <meta> etc. Then there is a starting container (<div class="container">).  Finally, the container DIV is closed and the included JS files section is included.  Following the JS is the closing </body> and </html> tags.  

- button.komodotool

This one works a little differently than the previous items.  The button setup within Foundation has multiple options, and capitalization is (obviously) important.

You'll be presented with 3 variables; the first is the size of the button--small, medium, or large.  The second is color--blue, white, red, black.  The third is button type--round or radius.

Once you've selected your variables (leave a field blank if you don't want to use that styling option), it dumps in the code.  

Example: I want a small blue button using radius.  Here is my output after entering "small", "blue", and "radius". 

<a href="#" class="small blue button radius">Button »</a>

- centered column.komodotool

Usage is identical to 1_column above, except this snippet adds the 'centered' class.  

- offset.komodotool

Usage is identical to 1_column above, except this snippet has two variables.  The first variable is # of columns for width of this column.  The second variable is your offset-by amount.  So, if you want to have six columns that are offset by two columns, you'd type "six" and "two" respectively.  

Output:

<div class="six columns offset-by-two">

</div><!--six Column offset by two-->

- _container.komodotool

Inserts:

<div class="container">

</div><!--container-->

- _row.komodotool

Inserts: 

<div class="row">
</div><!--row-->


+ Tables

I'm not going to type all of the files out. Here is the usage: 

The 'list01col.komodotool' file adds a one-column table.  Here is the output:

    <table>
            <thead>
                    <th>NAME</th>

            </thead>

    </table>

The list02col.komodotool' file inserts a two-column table.  Here is the output: 

    <table>
            <thead>
                    <th>NAME</th>
                    <th>NAME</th>
            </thead>

    </table>

The list(nn)ColRow.komodotool files give you your rows.  Remember that the 'col' and 'colRow' files are paired--so if you add a two-column table, you need to use the two-column row snippets!

Output (two column row): 

         <tr>
                <td>DATA</td>
                <td>DATA</td>
        </tr>

+ Tabs

This is also straightforward.  Choose the snippet based on how many tabs you want to have.  The snippet is set up to (a) be a contained tab set (you can remove 'contained' if you don't want that styling).  It also sets the first tab to be the active tab.  Again, simple to change.  The hrefs, obviously, point to nowhere.  

Output: 

				<dl class="contained tabs">
					<dd><a href="#nice1" class="active">NAME</a></dd>
					<dd><a href="#nice2">NAME</a></dd>
					<dd><a href="#nice3">NAME</a></dd>
				</dl>
